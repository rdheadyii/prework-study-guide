HTML

  • !DOCTYPE declaration: tells the browser what document type it should expect when it is opened.
  • html element: element that all the code is written in.
  • The head element contains information about the webpage.
    • title element: defines title of for the webapge, important for SEO.
    • style element: defines CSS styles associated with website.
    • base element: defines base URL for a webpage.
    • link element: connects an external resource to the HTML document.
    • meta element: defines metadata such as charset, description, keywords, author, and viewport.
    • script element: embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
    • div element: creates division in page.
    • span element: creates and inline division.
    • h1-6: level of heading in the text and should be used in order with h1 only appearing once on each page.
    • p element: paragraph or block of text; this will be used extensively to wrap most of text on webpages.
    • ul element: unordered list (bullet points)
    • ol element: ordered list (numbers, letters, roman numerals, etc.)
    • li element: list itmes
    • img element: contains info about images displayed of webpage.
    • src attribute: defines location of image.
    • alt attribute: text string that describes the appearance and functionality of the image.
    • br element: creates a line of empty space or line break.
    • a element: (anchor) creates links to the same webpage or other webpages.
    • href attribute: points to URl for the link.
    • header element: top of page and usually contains a nav menu and/or intro content.
    • main element: main body of the webpage.
    • footer element: appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.
    • !-- comments --: adds comments to the code that do not show on webpage.
    • section element: semantic element that define a section in a document that contains thematic content.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git revert SHA: reverts to previous version.
  • git add -a: adds all changes in current working branch to the staging area.
  • git commit -m "descriptive message": commits staged content as a new commit snapshot with a message of what was committed.
  • git pull origin main: receive version of code from the main branch of the repo.
  • git push origin: pushes the changes that are local to the GitHub branch.
